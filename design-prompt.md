# Промт для дизайнера: Т-Командировка

> Для ML-дизайнера / UI-разработчика
> Версия: 1.0 | 1 марта 2026

---

## КОНТЕКСТ ПРОЕКТА

Ты разрабатываешь дизайн **Т-Командировка** — AI-first корпоративная платформа управления деловыми поездками в экосистеме Т-Банка. Продукт должен выглядеть как premium B2B-сервис уровня Notion/Linear/Vercel, но с фирменным стилем Т-Банка и AI на первом плане.

---

## АНАЛИЗ РЕФЕРЕНСОВ (текущие конкуренты)

### Что я вижу на скринах (платформа «МАЙТИ» / Smartway)

**Экран «Командировки» (список):**
- Хедер: логотип «МАЙТИ», поисковая строка, иконки (чат, нотификации с бейджем «2», аватар)
- Табы: Мои командировки / Архив / Согласование командировок — простые pill-кнопки
- Empty state: иллюстрация (чемодан), текст «Командировки ещё не оформлены»
- Правый sidebar: FAQ-карточка, жёлтый блок обратной связи «Помоги нам стать лучше!»
- Кнопка CTA: «Создать командировку» (жёлтая, outline)

**Экран «Новая командировка» — шаг 1 «О поездке»:**
- Stepper: 1 О поездке → 2 Билеты и отель → 3 Подтверждение (жёлтый круг на активном)
- Поля: Город (dropdown) + Период (date range picker) на одной строке
- «+ Город» — возможность мультигородовой поездки
- Кто едет — dropdown с тегами (Калинин Петр ×)
- Цель — dropdown (Участие в проектах ЦУ)
- Название проекта — текстовое поле
- «+ Цель» — добавление нескольких целей
- Кнопки: «Продолжить» (жёлтая заливка), «Удалить черновик» (текстовая ссылка)
- Автосохранение: «Сохранено 01 марта 10:46:35»

**Экран «Новая командировка» — шаг 2 «Билеты и отель»:**
- Табы транспорта: Самолёт (radio active) · Поезд · Смешанный
- Блок «Нужен билет» (toggle on/off, жёлтый):
  - Переключатель: Туда-обратно / В одну сторону
  - Поля: Откуда → Куда, Даты (09.03 — 13.03), Вылет туда / Вылет обратно — по 2 dropdown в строке
  - «Найти билет» (жёлтая кнопка outline), toggle «Добавить багаж»
- Блок «Нужен отель» (toggle):
  - Поля: Название или локация, Заезд и выезд
  - Ссылка: «Посмотри корпоративные отели»
  - «Найти отель» (жёлтая outline)
- Блок: «Укажи почему выходные в командировке» — дата + dropdown «Выбери причину» + toggle «Весь день»
- Правый sidebar: «Чат по заявке» — виджет с фиолетовым градиентом, иконка чата

### Что ХОРОШО в текущем UX (забрать):
1. **Stepper 1-2-3** — понятная пошаговая навигация, не перегружает
2. **Toggle-блоки** «Нужен билет» / «Нужен отель» — progressive disclosure, юзер включает только то, что нужно
3. **Автосохранение** черновиков — не теряется прогресс
4. **«+ Город»** — поддержка мультигородовых маршрутов
5. **Чат по заявке** — контекстная поддержка прямо на экране
6. **Чистый, белый layout** — минимум визуального шума
7. **Жёлтый акцент** — фирменный, не перегружает

### Что ПЛОХО / наши точки улучшения:
1. **Нет AI-first подхода** — всё начинается с форм, а не с естественного языка
2. **Много полей сразу** — шаг 2 перегружен (билет + отель + причина выходных)
3. **Нет inline-результатов** — кнопка «Найти билет» уводит на отдельную страницу, нет мгновенной выдачи
4. **Нет AI-рекомендаций** — не подсказывает лучшую цену, не предлагает альтернативы
5. **Нет виртуальной карты** — нет финансового слоя в UI
6. **Нет OCR чеков** — нет камеры / загрузки чеков
7. **Empty state скучный** — просто «не оформлены», нет onboarding-подсказок
8. **Sidebar справа** малоинформативен — FAQ и обратная связь вместо аналитики/статусов
9. **Нет AI-инсайтов** по расходам, прогнозам, экономии
10. **Визуально устаревший** — стиль 2020-х, нет современных микроанимаций

## ЗАДАНИЕ ДЛЯ ДИЗАЙНЕРА

### Общая задача

Спроектировать полный UI-kit и ключевые экраны для платформы **Т-Командировка** в стиле Т-Банка. Платформа — AI-first: главная точка входа — разговорный AI-агент, а не формы. Классический форменный интерфейс (как у конкурентов) остаётся как fallback, но AI-чат — основной сценарий.

#### ЭКРАН 1: Главная (AI-First Landing)

**Цель:** Первый экран после логина. Точка входа — AI-чат.

**Layout:**
- Хедер (sticky, 56px): логотип «Т» + «Командировка», навигация (Главная / Командировки / Бронирование / Финансы / Аналитика), переключатель ролей (Сотрудник / Администратор), нотификации (бейдж), аватар с именем
- Hero-секция (по центру, max-width 600px):
  - Заголовок: «Куда летим?» (H1, 32px, bold)
  - Подзаголовок: «Опишите командировку — AI подберёт лучшие варианты» (Body, text-secondary)
  - **Онбординг-шаги** (3 карточки в ряд):
    - [1] Укажите направление, даты и бюджет
    - [2] Бот уточнит детали: багаж, трансфер, отель
    - [3] Получите варианты и забронируйте
    - Каждая карточка: белый фон, border, синий номер-кружок, text-secondary описание
  - **Что умеет бот** (3 карточки): иконка в сером квадрате 40×40 + заголовок + описание
  - **Поисковая строка AI** (центральный элемент, как на скрине AI Travel Copilot):
    - Высота ~52px, border 2px, radius 24px, shadow-sm
    - Placeholder: «Например: 1 билет в Казань, самолёт, 10 марта»
    - Слева: кнопка микрофон (transparent)
    - Справа: кнопка отправить (синий круг, ai-blue)
    - Focus state: border → ai-blue, shadow → ai-glow
  - **Quick scenarios** (pill-кнопки без иконок, radius 999):
    - «Билет в Казань на 10 марта»
    - «Командировка в Екатеринбург»
    - «Отель в Сочи до 5000₽»
    - «Авансовый отчёт»
    - Hover: border → ai-blue, bg → surface-ai
  - **Bottom tabs** (категории с иконками): Авиа, Отели, Поезда, Такси, MICE
- Секция «Мои командировки» (карточки поездок):
  - Каждая карточка: badge статуса (pill), маршрут (H3), даты (caption), chip-теги услуг, итого (bold)
  - Статусы: На согласовании (orange), Забронировано (green), В поездке (blue)

**Состояния:**
- Роль «Администратор» — заголовок меняется на «Управление командировками», quick scenarios меняются на административные

#### ЭКРАН 2: AI-чат (основной сценарий бронирования)

**Цель:** Полноценный чат с AI-ассистентом. Сюда пользователь попадает после отправки запроса.

**Layout:**
- Чат-контейнер (max-width 680px, центрирован)
- **Онбординг-сообщение бота** (первое сообщение, карточка с особым стилем):
  - Заголовок: «Т-Помощник» (bold)
  - Текст: «Привет! Опишите запрос — я подберу лучшие варианты.»
  - 3 шага (синие нумерованные кружки + текст):
    1. Направление, дата, количество билетов
    2. Я уточню детали: багаж, обратный, отель
    3. Покажу лучшие варианты — выберете и забронируете
  - Кликабельный пример запроса (серый блок, hover → border blue)
- **Сообщения пользователя:** синий фон (ai-blue), белый текст, radius 16px, top-right corner 4px
- **Сообщения бота:** белый фон, border, radius 16px, top-left corner 4px
- **Аватар бота:** 32×32, radius 10px, bg ai-blue/10, SVG-иконка внутри
- **Аватар юзера:** 32×32, radius 50%, bg brand-yellow, инициалы

**Inline-компоненты в ответах бота:**

1. **Кнопки уточнения (clarify buttons):**
   - Бот спрашивает: «Нужен обратный билет?»
   - Pill-кнопки: «Да, обратный нужен» / «Нет, только туда»
   - Стиль: border, radius 999, hover → border blue, selected → bg blue, color white
   - Бот последовательно спрашивает: обратный билет → багаж → отель (с завтраком?) → трансфер

2. **Карточки результатов (flight/hotel cards):**
   - Белый фон, border, radius 12px, padding 14px
   - Бейдж «Лучшая цена» (ai-blue bg/10, синий текст)
   - Строка: авиакомпания (bold) — цена (H3, bold, right-aligned)
   - Маршрут: SVO → KZN · 08:00—09:30 · Прямой · 1ч 30м (caption, text-secondary)
   - Теги: «В рамках политики» (green bg/10) / «Превышает лимит» (orange bg/10) / «Багаж 23 кг» (gray bg)
   - Кнопка «Выбрать» (yellow, full-width, radius 8px)
   - **Best card:** border → ai-blue, bg → #FAFCFF, бейдж сверху
   - Карточки кликабельны — по клику обводка синяя

3. **AI-инсайт:**
   - Блок: bg ai-blue/8, border-left 3px ai-blue, radius 12px, padding 12px
   - Текст: «AI-совет: цена на S7 может вырасти на ~12% завтра» (font-size 13px, color ai-blue)

4. **Сводка бронирования (summary):**
   - bg surface-secondary, radius 12px, padding 14px
   - Строки: label — value (justify between)
   - Итого: border-top, font bold, font-size 15px
   - Кэшбэк: green, font-size 12px

5. **Кнопки действий:**
   - Primary: «Забронировать всё» — bg yellow, radius 10px
   - Secondary: «Изменить» — border, transparent bg
   - Состояния primary: default → loading (текст «Бронирую...», bg blue) → success («Забронировано», bg green)

6. **Typing indicator:** три точки, animation blink, bg ai-blue

- **Фиксированный input внизу экрана:** sticky bottom, blur-backdrop, border-top, внутри — тот же search-wrap

#### ЭКРАН 4: Бронирование (классический fallback)

**Цель:** Форменный поиск для тех, кто предпочитает ручной режим. Повторяет логику МАЙТИ, но улучшенную.

**Layout:**
- Tabs: Авиа / Поезда / Отели / Такси (внутри pill-контейнера с bg, active → white + shadow)
- Форма (max-width 640px, центрирована, white card, radius 20px, shadow-md):
  - Grid 2 колонки: Откуда/Куда, Туда/Обратно (date), Сотрудник (select), Командировка (select)
  - Кнопка «Найти» (yellow, full-width, radius 10px)

**Улучшения vs МАЙТИ:**
- НЕ toggle-блоки, а чистая форма одного типа (переключается табами)
- После нажатия «Найти» — результаты появляются INLINE под формой (а не redirect)
- AI-инсайт под результатами: «Купите сейчас — завтра +18%»

### СЦЕНАРИИ ДЛЯ ДЕМОНСТРАЦИИ В ПРОТОТИПЕ

**Сценарий 1 — Сотрудник бронирует командировку:**
1. Главная → пишет «1 билет в Казань, самолёт, 10 марта»
2. AI: онбординг-сообщение → затем уточняет: обратный? → багаж? → отель? → трансфер?
3. AI: показывает 3 карточки рейсов (best выделен) + 2 отеля + трансфер
4. AI: сводка с итого + кэшбэк
5. Кнопка «Забронировать всё» → loading → success → подтверждение с виртуальной картой

**Сценарий 2 — Сотрудник отправляет чеки:**
1. Главная → пишет «Авансовый отчёт»
2. AI: предлагает загрузить фото / ввести вручную / показать текущий отчёт
3. Фото чека → OCR → сумма, дата, НДС распознаны → добавлено в отчёт

**Сценарий 3 — Администратор согласовывает:**
1. Переключатель роли → Администратор
2. Список «Ожидают согласования» — карточки с кнопками Согласовать / Отклонить
3. Аналитика — AI-инсайты по перерасходам
